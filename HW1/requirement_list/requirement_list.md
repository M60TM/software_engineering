# Requirement List

팀 코드: 53607

## Use Case 분배
actor로는 사용자, 회원, 관리자를 도출할 수 있었고, main use case는 총 12개 도출되었으며, extend를 포함하면, 14개의 use case가 도출됨. 이를 비슷한 UI나 동일 actor의 기능을 기준으로 아래와 같이 분배함.
- 장의진: 설문 응답, 응답 설문 조회, 응답 취소, 응답 수정
- 천병선: 설문 검색, 설문 상세 정보 조회
- 김기현: 설문 등록, 설문 조회, 설문 삭제, 설문 통계
- 황재민: 회원 가입, 회원 탈퇴, 로그인, 로그아웃

---

| No.  | Requirement | Use Case(s) |
|---|---|---|
| 1. | 시스템을 이용하려면 사용자는 회원 가입을 해야 한다. 회원의 필수 입력 정보는 ID, 비밀번호, 이름, 전화번호, 이메일 등이다. | 회원 가입 |
| 2. | 회원은 언제든 탈퇴할 수 있으며, 탈퇴 시 모든 이용 권한과 데이터가 삭제된다. | 회원 탈퇴 |
| 3. | 관리자와 회원은 ID와 비밀번호로 로그인하며, 관리자는 주어진 ID(admin)와 비밀번호 (admin)으로 로그인한다. | 로그인 |
| 4. | 회원은 언제든 로그아웃할 수 있다. | 로그아웃 |
| 5. | 회원은 조건에 맞는 설문을 검색할 수 있다. 검색을 위해 현재 진행 중인 설문 리스트, 향후 진행 예정인 설문 리스트, 종료된 설문 리스트 중 하나를 선택한 후 키워드를 입력한다. 검색 결과로 조건에 맞는 설문 리스트가 출력된다. | 설문 검색 |
| 6. | 회원은 검색된 설문 리스트 화면에서 특정 설문을 선택하면 새로운 화면에서 상세 정보 화면을 볼 수 있다. 해당 설문 상세 정보 화면에서는 설문 제목, 설문 설명, 설문 문항, 응답 항목, 설문 시작 시각, 설문 마감 시각 등이 출력된다. 현재 진행 중인 설문인 경우에는 응답 버튼이 활성화된다. | 설문 상세정보 조회 |
| 7. | 회원이 응답 버튼을 누르면 응답 화면으로 이동하여 설문에 응답할 수 있으며 응답 버튼을 눌러서 완료한다. | 설문 응답 |
| 8. | 회원은 자신이 응답한 모든 설문 정보를 한 화면에서 조회할 수 있다. 각 응답한 설문에 대해 설문 제목, 응답 내용, 응답 일시 등이 출력된다.<br>(응답 취소) 또한, 현재 진행 중인 설문에 대해서는 응답을 취소할 수 있다. 단, 설문이 종료된 경우에는 응답 내용을 수정하거나 취소할 수 없다. | 응답 설문 조회 |
| 9. | 회원은 현재 진행 중인 설문에 대해서는 자신이 제출한 응답을 수정할 수 있다. 이때, 설문 수정 화면에는 가장 최근에 수정한 날짜가 출력되어 있으며 수정 버튼을 눌러서 완료한다. | 응답 수정 |
| 10. | 관리자는 설문 정보를 등록할 수 있다. 설문 등록 시 입력 정보는 설문 제목, 설문 설명, 설문 문항, 응답 항목, 설문 시작 시각, 설문 마감 시각 등이다. | 설문 등록 |
| 11. | 또한, 관리자는 등록된 모든 설문 리스트를 조회할 수 있다. 이때 설문 리스트 조회 화면에서 원하는 설문 항목을 선택하면 상세 내용을 팝업 창으로 볼 수 있다.<br>(설문 삭제) 또한, 설문 리스트 조회 화면에서 특정 설문 항목을 선택해서 삭제할 수 있다. | 설문 조회 |
| 12. | 관리자는 설문 응답 통계 정보를 조회할 수 있다. 검색 단위를 최근 1주일, 1개월, 1년 중에서 선택하면 그 기간 동안 전체 설문 수, 종료된 설문 수, 총 응답 수가 출력된다. | 설문 통계 |
